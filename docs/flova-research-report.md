# Flova.ai 竞品研究报告 — AstraAutoEx 改进方案

> 基于 2026-04-15 实际浏览 flova.ai 导演模式的研究

## 一、Flova.ai 核心架构

### 1. Chat-First 工作流（最大差异）
Flova 的核心理念：**AI 助手 IS 主界面**。用户不是点击按钮执行操作，而是在对话中告诉 AI 要做什么，AI 自动执行整个管线。

- 首页输入框 = AI 对话入口（不是表单）
- 底部工具栏：+ | 模型选择 | Skill | 元素
- 热门提示词卡片一键填充（导演模式、灵感模式、广告模式等）

### 2. 分屏布局
```
┌────────────────────────┬──────────────┐
│  故事板 / 时间线 (左)    │  AI 助手 (右)  │
│  - 关键元素（角色卡）     │  - 聊天对话    │
│  - 分镜（Shot 列表）     │  - 进度步骤    │
│  - 视频/音频预览        │  - 积分消耗    │
│                        │  - 评分反馈    │
└────────────────────────┴──────────────┘
```

### 3. Agent 执行步骤
每个生成任务显示清晰的步骤：
1. ✅ Agent 分析完成
2. ✅ 搜索信息完成
3. ✅ 资产配置完成
4. ✅ 编写提示词完成
5. ⏳ 生成素材中...

### 4. 资产管理
- **关键元素**：角色有多角度图片（正面/侧面/背面），标注模型名称
- **分镜 (Shot)**：每个 Shot 有详细描述 + 镜头运动 + 音频对齐标注
- **Visual Assets / Audio Assets**：可展开/收起的资产组，显示积分消耗

## 三、高优先级改进项（按影响排序）

### P0: 修复 AI 助手（立即）
1. **修复工作区面板消息回路** — send → handle_info → 返回回复到面板
2. **添加项目上下文** — 把当前故事/角色/场景注入 system prompt
3. **明显的切换按钮** — 固定在右下角的浮动按钮，带动画

### P1: AI 助手增强（短期）
4. **Agent 执行步骤 UI** — 每步操作显示：分析→搜索→配置→生成
5. **积分消耗实时显示** — 每次 AI 调用旁显示 token 消耗
6. **内联媒体预览** — 在助手对话中直接显示生成的图片/视频

### P2: 功能增强（中期）
7. **对话历史持久化** — 保存到数据库，刷新不丢失
8. **Skill 系统** — 预设技能卡片（导演模式、灵感模式等）
9. **元素引用** — @角色名 @场景名 语法，自动注入上下文
10. **质量评分** — 每次生成结果旁显示五星评分

### P3: UI 对齐（长期）
11. **暗色主题** — Flova 默认深色
12. **时间线视图** — 可视化编辑面板顺序+时长
13. **多角度角色图** — 一次生成正面/侧面/背面

## 四、实施计划

**本轮迭代重点（P0）：**
1. 修复 AI 助手消息回路 → 让对话真正可用
2. 注入项目上下文 → 让 AI 了解当前项目
3. 浮动切换按钮 → 让用户随时调出助手
4. 面板滑入/滑出动画 → 流畅的交互体验

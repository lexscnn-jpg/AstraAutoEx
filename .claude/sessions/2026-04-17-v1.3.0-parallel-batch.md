## A: 流式 LLM 输出（我串行做）

**核心思路**：原 Next.js 用 SSE stream (`story-to-script-stream`)，Phoenix 用 `Phoenix.LiveView` 的 message passing + Task.Supervisor 后台 stream。

**新模块** `lib/astra_auto_ex/ai/llm_streamer.ex`:
```elixir
LLMStreamer.stream_to(subscriber_pid, stream_id, user_id, provider, request)
# Emits messages:
#   {:llm_chunk, stream_id, text}
#   {:llm_done, stream_id, full_text}
#   {:llm_error, stream_id, reason}
```

**双路径**：
- Provider 有原生 `chat_stream/2`（apiyi/google）→ 消费 SSE chunks 直接 forward
- Provider 无流能力（minimax/ark/fal/...）→ 回退 `chat/3` 然后把完整响应按 50-字符切片模拟流（保证 UI 体验一致）

**集成**：`home_live.ex` 的 AI 写作弹窗从 `Helpers.chat` 切到 `LLMStreamer.stream_to`，加 3 个 handle_info 分别处理 chunk/done/error。用户下次点"AI 写作"立刻看到 token 实时吐出。

**supervisor**：`LLMStreamer.task_supervisor_child_spec` 挂到 application.ex children。

## B: 撤销/撤回（我串行做）

**现状**：Panel schema 有 `image_history :map` 字段但没任何使用。原 Next.js 有 10+ undo endpoints。

**新增 Production 函数**:
```elixir
update_panel_image_with_history(panel, new_url)
# 自动把旧 image_url 推入 image_history.revisions[]
# 限制最近 10 条

undo_panel_image(panel)
# 弹出 revisions 最后一条回到 image_url
# 返回 {:ok, panel} 或 {:error, :no_history}
```

**接入点**：
- `image_handlers.ex` 所有 `Production.update_panel(panel, %{image_url: ...})` 替换为 `update_panel_image_with_history/2`
- `workspace_live/show.ex` 加 `handle_event("undo_panel_image", %{"panel-id" => id}, socket)`

**好处**：每次图生成自动追踪，用户点撤销即刻回滚，不用额外存储/迁移。

## C: 批量 ZIP 导出（我串行做）

**新 controller** `lib/astra_auto_ex_web/controllers/export_controller.ex`:
- `project_download(conn, %{"project_id" => id, "kind" => "images"|"videos"|"voices"})`
- 聚合所有 panels 的 image_url/video_url 或 voice_lines 的 audio_url
- 用 Erlang 内置 `:zip.create/3` 打包

**双路径**：
- 总大小 < 100MB → 内存 ZIP（快）
- > 100MB → 写 temp file（不占内存）

**源 URL 处理**：
- `/uploads/xxx` → 读本地文件
- `http(s)://...` → Req.get 下载

**UI 集成**：compose 阶段（AI 剪辑页面）加 3 个按钮：📷图片ZIP / 🎬视频ZIP / 🎙️配音ZIP，紧跟 Recompose 按钮。

**安全**：`Projects.get_project!(id, user_id)` 强制校验用户只能下载自己的项目。

## D: Billing Ledger（agent worktree 产出）

**Agent 产出报告关键点**：

**4 核心 API**（带 @spec）:
```elixir
Ledger.topup(user_id, amount, meta)     # 充值
Ledger.freeze(user_id, task_id, amount)  # 任务前冻结
Ledger.claim(task_id, actual_amount)     # 任务成功 → 按实付扣减
Ledger.release(task_id)                  # 任务失败 → 全额退回
```

**并发安全设计**：
- 每次写路径 `SELECT ... FOR UPDATE` 锁 user_balances 行
- 2×7 单位 freeze 对 10 余额的竞争测试：只一个成功，另一个 `:insufficient_balance`

**claim 按实付封顶**：
- 如果 actual_amount > freeze.amount → 只扣 freeze.amount（不 overcharge）
- 多余部分退 balance

**审计流水**：每次 topup/freeze/claim/release 都写 balance_transactions 行（balance_after 快照 + metadata JSONB）

**向后兼容**：保留旧函数名 `freeze_balance` / `confirm_charge` / `rollback_freeze` 作为 alias。

**测试**：32 个 ledger tests（covering: read helpers / topup lifecycle / freeze edge cases / claim over/under / release / 完整生命周期 / 并发竞争）

**Ledger 覆盖率 84.76%**，schema 三个 100%。

**commit**: `edaaa98 feat(billing): 补齐 Ledger freeze/claim/release/topup 结算闭环`

## E: OAuth Google+GitHub（agent worktree 产出）

**Agent 产出报告关键点**：

**依赖选型**：`assent` 而非 `ueberauth`
- assent 是函数式（非 plug-middleware）
- 25+ 内置 strategy（包括 Google OIDC + GitHub OAuth2）
- 不需要 per-provider hex package
- ueberauth 需要 `ueberauth_google` + `ueberauth_github` 分别装

**数据模型扩展**：
- users 表加 `oauth_provider` (string, null) + `oauth_uid` (string, null)
- **Partial unique index** WHERE oauth_provider IS NOT NULL AND oauth_uid IS NOT NULL — 密码用户（两列 null）不受 unique 约束影响

**核心流程** (`find_or_create_user_from_oauth/2`)：
1. 如果 `oauth_uid` 已存在 → 返回已有用户
2. 如果 email 匹配 existing password 用户 → **link**（保留密码，同时允许 OAuth 登录）
3. 否则新建用户 auto-confirmed（provider 已验证邮箱）

**路由**：
```
GET /auth/:provider             → request (OAuth redirect)
GET /auth/:provider/callback    → callback (find_or_create + log in)
```

**UI**：login 页加 Google + GitHub 2 个品牌按钮（glass-morphism 风格 + SVG icons）

**Setup 文档**（已写进 README）：
```
export GOOGLE_CLIENT_ID="..."
export GOOGLE_CLIENT_SECRET="..."
export GITHUB_CLIENT_ID="..."
export GITHUB_CLIENT_SECRET="..."
```
callback URLs:
- `http://localhost:4000/auth/google/callback` (Google Cloud Console)
- `http://localhost:4000/auth/github/callback` (GitHub Developer Settings)

**测试**：13 个 scenarios 覆盖 request + callback + find_or_create 的所有分支（新用户/已存在 OAuth/ email match password/ 跨 provider email match/ 未提供 email 错误）

**commit**: `22064b6 feat: OAuth Google/GitHub (assent + 13 tests)`

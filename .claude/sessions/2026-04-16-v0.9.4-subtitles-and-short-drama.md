## 按步骤的技术记录

### S1：LipSync audio_url schema 修复

**症状**：video_handlers.ex:235 `audio_url = payload["audio_url"] || panel.audio_url` —— Panel schema 没 `:audio_url` 字段。下次 lip_sync 真跑会 KeyError。

**修**：新增 `Production.voice_line_for_panel/1`，按 `panel_id` 或 `matched_panel_id` 查关联 VoiceLine，取它的 `audio_url`。LipSync handler 改为：

```elixir
audio_url =
  payload["audio_url"] ||
    case Production.voice_line_for_panel(panel.id) do
      %{audio_url: url} when is_binary(url) and url != "" -> url
      _ -> nil
    end
```

## 技术观察

**subtitle filter 在 Windows 上的坑**：
- 必须用正斜杠（即使路径原本是反斜杠）
- 冒号必须转义（`C:/` → `C\:/`）
- 这两点一起应用后才能让 ffmpeg 认出路径

**Windows 热重载**：重启仍然是唯一可靠方式。本轮重启约 3 次。

**动态切换 preview server** 带来的坑：每次 preview_stop + start 会改 serverId，但我 todo 里引用旧 id 时不会失效。serverId 应该在需要时 preview_list 重取。

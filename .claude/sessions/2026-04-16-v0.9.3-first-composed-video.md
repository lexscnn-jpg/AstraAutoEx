## R3：Compose 真实合成（本轮 90% 时间）

这是 **5 个独立 P0/P1 bug 堆叠** 的排雷过程：

### Bug 1：Storyboard schema 不一致
- `Enum.sort_by(& &1.sort_order)` → `KeyError :sort_order not found`
- Storyboard 实际字段：`episode_id / panel_count / storyboard_text_json / image_history`
- 改为 `& &1.inserted_at` with `NaiveDateTime` 比较器
- Panel 同样问题：用 `:panel_index` 非 `:sort_order`
- Panel 还根本没有 `:audio_url` 和 `:duration` 字段（我之前写了这些 Map 键，会在真实使用时 crash）

### Bug 2：Elixir `or` 严格类型
- `(p.video_url && p.video_url != "") or (p.image_url && p.image_url != "")`
- 当 p.video_url 是 nil：`nil && "..."` 返回 nil
- `nil or ...` → **"expected a boolean on left-side of or"**
- 改为 `is_binary(...) and ... != ""` + `or`（两边都是 boolean）

### Bug 3：输出路径目录未创建
- `storage_key` 从 `Provider.generate_key("compose", "mp4", project_id: ..., media_type: "video")`
- 实际值含 `projects/13/video/compose-....mp4` 嵌套路径
- 代码 `File.mkdir_p!(Path.join(upload_dir, "compose"))` 只创建到顶级
- FFmpeg 报 `No such file or directory`
- 改为 `File.mkdir_p!(Path.dirname(output_path))`

### Bug 4：无图像→视频转换能力
- 项目 13 的 panel 都只有 image_url 没 video_url（没走过 video_panel）
- 原 compose handler 过滤后 panel_videos 为空 → 直接返回"No panel videos available"
- 这阻止任何"图像直出短片"的用例
- 修复：新增 `FFmpeg.image_to_video/3`，用 `-loop 1 -i image -f lavfi -i anullsrc -t N -c:v libx264 -tune stillimage -pix_fmt yuv420p -vf scale=偶数 -c:a aac -shortest`
- VideoCompose handler 里：无 video_url 的 panel 先调 image_to_video 生成临时 mp4，再传给 compose

### Bug 5：filter_complex 在静态片段上失败（未根治，但绕过）
- compose 第一次 "completed 100%" 但 result 含 error："FFmpeg completed but output file not found"
- 真因被 `run_ffmpeg` 吞了（return value 被丢弃）
- 修 run_ffmpeg：显式 `case` 捕获 `{:ok, _}` / `{:error, reason}`，错误进 task.error_message
- 再跑 → error 是 ffmpeg exit -2 + 被截断
- 直接加 simple_concat fallback：`compose` 失败时自动走 concat demuxer，无 xfade 无字幕，只是拼起来
- concat demuxer 用 `-c copy` 先尝试，失败则 `reencode_concat` 用 libx264/aac 重编
- 这次成功

### Bug 6：.mp4.mp4 双重扩展名
- storage_key 已含 ".mp4"，代码又 `"#{storage_key}.mp4"`
- 输出文件名变 `compose-....mp4.mp4`
- 修：`if String.ends_with?(storage_key, ".mp4"), do: storage_key, else: storage_key <> ".mp4"`

## 技术观察

**热重载在 Windows 上不可靠**：
- 改 Elixir 模块后，preview_start 的 server 的 code_reloader 经常不 pick up
- 每次都得 preview_stop → preview_start 才能看新代码效果
- 这一轮重启了至少 4 次
- **Workaround**：每次重要改动后直接重启，别信 hot reload

**Req vs Finch**：
- 我第一版写了 `Finch.request/2` 但 Application 没注册 named pool
- 换 `Req.get/2`（已在依赖里）就 work 了
- Finch 更高性能但要 pool 配置；Req 零配置

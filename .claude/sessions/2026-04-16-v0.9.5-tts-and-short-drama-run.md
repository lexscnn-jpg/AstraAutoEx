## 细节

### T1-T2 真实 TTS

**前置修复**：
- MiniMax provider 的 `text_to_speech/2` 在 line 198 `get_in(resp, ["data", "audio", "data"])` 当 resp 是 binary（非 map）时触发 `Access.get/3` 崩溃
- 修：改用 pattern match `case resp do %{"data" => %{"audio" => %{"data" => h}}} -> ...; _ -> ...`
- MiniMax TTS 返回 hex-encoded audio payload，加 `save_hex_as_wav/2`：Base.decode16 + File.write + 返回 `/uploads/voice/tts-<unique>.wav`

**结果**：
```
Task: completed 100%
audio_url: /uploads/voice/tts-24.wav
File: 281K WAV, 32kHz mono PCM_S16LE, duration=4.469844s
```

然后 5 条 voice_lines 全部 TTS 完成（8-16 秒内）。

### T3 有声视频合成

之前 compose 只处理视频流（图像→静态 mp4→concat），没混音。需要：
1. 从 voice_lines 提取 audio_path + start_time
2. 传给 FFmpeg filter_complex

**新增** `build_voice_segments/1`：
- 过滤有真实 audio_url 的 voice_lines（跳过 placeholder://）
- 按 line_index 排序
- 累加 cursor（用 audio_duration 或默认 4.0s）
- 输出 `%{audio_path, start_time}` list

**resolve_local_audio/2**：`/uploads/voice/x.wav` → `priv/uploads/voice/x.wav` 本地文件路径（FFmpeg 需要）

结果：
- 新视频：2.6MB（vs 纯图片版 2.2MB，多 0.4MB 就是 audio）
- ffprobe 确认两路 stream：H.264 video + AAC 44.1kHz stereo audio
- 时长 48.2s 不变，但现在**能听到中文旁白**

## 技术坑点

**1. 手动 JS `.value =` 不触发 phx-change**
- Phoenix LiveView 的 phx-change 只监听真实 user input 事件，programmatic 改 select.value 不算
- 要测切换得用 `liveSocket.getViewByEl(main).pushEvent('event', el, 'select_project', payload)`

**2. MiniMax TTS 响应结构**
- v2 返回两种可能：(a) 同步带 hex audio (b) 异步带 file_id 需 retrieve_file
- 两个分支在 provider 里都实现了，但 (a) 分支用 get_in 崩
- pattern match 更安全（拿不到字段返回 nil，不会 raise）

**3. voice_lines 必须有 audio_url 才进字幕生成器**
- v0.9.4 遇到过，这次知道了要么填真实 URL 要么 placeholder
- build_voice_segments 过滤 placeholder:// 前缀跳过（只留真实文件）

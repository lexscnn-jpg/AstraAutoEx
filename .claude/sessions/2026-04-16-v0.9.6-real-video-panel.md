## 4 个连环 P0

实现这个目标过程中撞到 4 个独立的 P0 bug：

### P0-1: apply_model_suffix 给 MiniMax 加了无效后缀

- 原代码对所有 provider 都调用 apply_model_suffix，aspect_ratio "16:9" 触发 `-landscape`
- `MiniMax-Hailuo-2.3-landscape` 不被 MiniMax 接受 → "invalid params, incorrect model param input"
- 修：只对 provider == "apiyi" 应用（这是给 VEO 模型的）

### P0-2: first_frame_image 字段名不匹配

- VideoPanel 传 `:image_url`
- MiniMax provider `put_if(body, "first_frame_image", Map.get(request, :first_frame_image))` 读 `:first_frame_image`
- 结果 image URL 丢失
- 修：handler 同时设 `:image_url` 和 `:first_frame_image`（保持 API易/ARK 兼容）

### P0-3: TaskRunner 吞掉异步任务状态

- MiniMax video_generation 返回 `{:ok, %{external_id: "MINIMAX:VIDEO:xxx", task_id: ...}}`（异步）
- VideoPanel handler 返回 `{:ok, result}`，TaskRunner 立刻 `Tasks.mark_completed` → task status=completed
- AsyncPollWorker 查 `status="processing" AND external_id != nil` 找不到这个 task，不轮询
- video_url 永远不会写入
- 修：给 TaskRunner 加 `{:async, _result}` 返回值，保留 processing 状态。VideoPanel handler 对应改走 `{:async, result}` 分支

### P0-4: AsyncPollWorker.atomize_keys/1 对非 map 崩

- `atomize_keys` 只 match `is_map(map)`，但 user_preferences.provider_configs 的某个 value 不是 map（可能是 string 或 nil）
- FunctionClauseError，poll 每 15 秒崩一次
- 修：加 catch-all 子句 `defp atomize_keys(other), do: other`

## MiniMax 模型探测（价值数据）

写了 probe 脚本测 MiniMax 视频生成支持哪些模型，证据如下：

| 模型 | 结果 |
|---|---|
| `MiniMax-Hailuo-2.3` | ✅ 接受（无图也 OK，T2V 模式） |
| `MiniMax-Hailuo-02` | ❌ token 计划不支持 "MiniMax-Hailuo-02-6s-768p" |
| `I2V-01` | ❌ 需要 first_frame_image |
| `I2V-01-Director` | ❌ 需要 first_frame_image |
| `I2V-01-live` | ❌ 需要 first_frame_image |
| `S2V-01` | ❌ 需要 subject_reference |

结论：**Hailuo-2.3 是当前订阅 token 下最佳选择**。I2V-01 系列需要显式传 first_frame_image，对图生视频场景更精确但需要 handler 修正（现已修）。

## 短剧 build_bindings + 串联（U1+U2）

**问题**：v0.9.5 跑 sd_topic_selection 时 LLM 返回"都市情感逆袭"但 topic_keyword 是"现代悬疑特警"——因为 bindings 根本没读 topic_keyword。

**修 U1**：sd_topic_selection bindings 加 `topic_keyword: payload["topic_keyword"] || payload["input_text"] || ""`

**修 U2**：新增 `load_previous_result/2` 从 DB 查最近完成的 prev_type task 的 result.raw：

```elixir
defp load_previous_result(project_id, prev_type) when is_integer(project_id) do
  q = from(t in AstraAutoEx.Tasks.Task,
    where: t.project_id == ^project_id and t.type == ^prev_type and t.status == "completed",
    order_by: [desc: t.finished_at], limit: 1)
  case AstraAutoEx.Repo.one(q) do
    %{result: %{"raw" => raw}} when is_binary(raw) -> raw |> String.slice(0..5999)
    _ -> ""
  end
end
```

然后在 sd_story_outline / sd_character_dev / sd_episode_directory 里用 `payload["x"] || load_previous_result(project_id, prev_type)`。

**效果**：用户只需点 "运行" 按钮，下游 step 自动读上游已完成 step 的 LLM 输出作上下文，不需要手动拼 payload。

## 技术观察

**热重载在 atomize_keys 修复上第一次生效**：其他 handler 修改要重启，但 AsyncPollWorker 是一个常驻 GenServer，代码热加载后新 poll 循环直接用新逻辑。这次修完 `atomize_keys` 没重启就自动跑通了（task 从 processing 变 completed）。

**MiniMax 视频生成耗时**：真实 I2V 任务约 **3-6 分钟完成**（20 秒等到 external_id，后台 MiniMax 渲染 3-6 分钟，AsyncPollWorker 15 秒一 poll 所以检测到完成会有 0-15 秒延迟）。
